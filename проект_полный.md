## 1. Название проекта

**Короткое название:** Agro-IoT Cattle Health Monitoring System (Система мониторинга здоровья скота Agro-IoT)

**Развёрнутое название:** Комплексная система непрерывного контроля физиологического состояния, поведенческих параметров и производительности крупного рогатого скота на основе биосенсоров, RFID-меток и алгоритмов машинного обучения для оптимизации доходности молочных ферм

**Номер группы:** 06-331

**Авторы проекта:**
Черепанов Сергей, Юсупова Элеонора

## 2. Целевая аудитория

### Основная целевая аудитория:

**Молочные фермы среднего и крупного масштаба** (100–1000 голов)

**Географический фокус:** Центральный, Поволжский, Уральский федеральные округа РФ (где сосредоточено 65% молочного скотоводства)

### Демографический и социально-экономический портрет:

| Параметр | Описание |
|----------|---------|
| **Тип хозяйства** | ООО, ЗАО, сельскохозяйственные кооперативы, крупные фермерские объединения |
| **Размер хозяйства** | Стадо 100–500 голов (оптимально 250–350) |
| **Уровень доходов** | Среднемесячный оборот ≥ 5 млн. руб. на полный цикл производства молока |
| **Уровень образования руководства** | Среднее профессиональное (ветеринарная, зоотехническая подготовка) и выше |
| **Технологический уровень** | Хозяйства, готовые к цифровизации: наличие интернета, опыт работы с современным оборудованием |
| **Возрастная демография персонала** | Операторы и менеджеры ферм в возрасте 35–55 лет; младший персонал 25–40 лет |

### Вторичная аудитория:

- **Ветеринарные клиники и консультанты** (для платных консультаций по результатам мониторинга)
- **Станции по борьбе с болезнями животных** (СББЖ) — возможность интеграции с государственными сервисами эпидемиологического контроля
- **Агрохолдинги и интегрированные системы производства** (от первичного животноводства до переработки молока)

### Преимущества для целевой аудитории:

1. **Экономическая выгода:** Повышение продуктивности на 10–15% и снижение затрат на лечение на 30–40% за счет ранней диагностики заболеваний
2. **Оптимизация труда:** Автоматизация рутинных операций контроля, сокращение ручного наблюдения за животными
3. **Соответствие стандартам:** Соответствие требованиям программ государственной поддержки цифровизации АПК (ГЦИФР, NTI «Technet»)
4. **Конкурентное преимущество:** Возможность экспортировать молоко высочайшего качества благодаря сокращению случаев скрытых инфекций
5. **Безопасность животных:** Минимизация использования антибиотиков, улучшение благосостояния животных

## 3. Инновация в проекте

### Что нового вносит данная система?

Система представляет собой **первый комплексный российский IoT-решение для непрерывного мониторинга здоровья КРС**, объединяющее:

1. **Многоуровневую биосенсорную архитектуру:**
   - Ушная RFID-бирка с датчиками температуры и акселерометром (регистрация осанки, лежания, движений)
   - Смарт-ошейник с акустическим датчиком жвачки и GPS-трекером
   - Интегрированный сенсор в доильный стакан (электропроводность, жирность, белок молока в реальном времени)

2. **Гибридную архитектуру связи (LoRa + Wi-Fi + BLE)**, позволяющую работать в условиях слабого покрытия типичной российской фермы

3. **Отечественные ML-алгоритмы** прогнозирования заболеваний (мастит, кетоз, тепловой стресс), обученные на популяциях российских пород коров (черно-пестрая, красно-пестрая)

4. **Полное соответствие российскому законодательству** по защите данных (152-ФЗ), расположение серверов в РФ

## 4. Функционал. Описание возможностей и технической реализации

### Основные функции системы:

#### 4.1 Сбор биометрических данных в реальном времени

**Ушная бирка (RFID UHF + датчики):**
- Температура тела (диапазон 36–41°C, погрешность ±0,3°C)
- Трехосевой акселерометр: регистрирует позу животного (стоит/лежит), интенсивность движений
- RFID-модуль UHF 865–868 МГц для идентификации животного на расстоянии 5–8 метров
- Батарея: Li-Pol 500 мАч (автономность ≥18 месяцев в режиме передачи данных каждые 5 минут)

**Смарт-ошейник:**
- Акустический датчик жвачки: регистрирует частоту и характер жевательных движений (норма 40–80 движений в минуту)
- GPS-трекер (опционально): фиксирует перемещение животного по территории фермы, распределение времени в разных секторах
- 9-осевой акселерометр + гироскоп: детальное отслеживание активности, выявление хромоты
- Батарея: Li-Pol 800 мАч (автономность ≥24 месяца)

**Интегрированный сенсор молока (в доильный стакан):**
- Электропроводность молока (норма 4–6 мСм/см; повышение указывает на мастит)
- Процентная доля жира и белка (инфракрасная спектроскопия)
- Объем надоя (ультразвуковой датчик в резервуаре)
- Загруженность доильного аппарата и время процедуры

#### 4.2 Передача и обработка данных

**Локальный шлюз передачи данных:**
- Получает сигналы от 50–100 устройств одновременно по протоколу **LoRa** (дальность до 2–3 км, пропускная способность достаточна для ≈10 тыс. событий/час)
- Резервный Wi-Fi (для зон с хорошим сигналом роутера)
- Локальное кэширование данных на SD-карте (буфер 7–14 дней при отсутствии интернета)
- Шифрование данных на уровне приложения (AES-256)

**Облачная платформа (Backend на базе FastAPI + PostgreSQL):**
- Прием и нормализация потока данных в реальном времени (MQTT-бrokering)
- Трансформация сырых показаний в медицинские/зоотехнические показатели
- Архивирование в TimescaleDB (оптимизирована для временных рядов, срок хранения ≥3 лет)

#### 4.3 Интеллектуальная диагностика и прогнозирование

**Модуль машинного обучения:**

1. **Классификация заболеваний** (Random Forest + XGBoost):
   - **Субклинический мастит:** выявление по изменению электропроводности молока, температуры, снижению жвачки
   - **Кетоз (кетонемия):** детектирование по комбинации показателей — снижение молочного жира, снижение активности, изменение электропроводности
   - **Тепловой стресс:** сочетание повышенной температуры, учащения дыхания, снижения жвачки в жаркий период
   - **Хромота:** анализ паттернов ускорения из датчика ошейника, асимметрия в опоре

2. **Прогнозирование качества молока** (LSTM — долгосрочная кратковременная память):
   - Предсказание динамики жирности и белковости на 1–2 недели вперед
   - Идентификация животных, молоко которых рискует снизиться в классности

3. **Выявление аномалий поведения** (K-means кластеризация + Isolation Forest):
   - Обнаружение непредвиденных изменений в режиме жвачки, активности, лежания
   - Ранний сигнал о начинающемся воспалении или стрессе до появления явных клинических признаков

#### 4.4 Рекомендации и управление

**Автоматические рекомендации:**
- Уведомления фермеру о необходимости клинического осмотра животного
- Рекомендации по кормлению и управлению микроклиматом коровника
- Напоминания о вакцинациях и профилактических обработках на основе истории животного

**Аналитический дашборд:**
- Обзор здоровья стада (процент животных в норме/риске/тревоге)
- Индивидуальные карточки коров с полной историей показателей
- Сравнительный анализ продуктивности по периодам (лактационные циклы)
- Экспорт отчетов в Excel и интеграция с 1С (бухгалтерское учёт, расчеты себестоимости)

### 4.5 Технологии и техническая реализация

#### Аппаратная архитектура:

```
┌─────────────────────────────────────────────────────────────┐
│         Биосенсорные устройства на животном                 │
├─────────────────────────────────────────────────────────────┤
│  Ушная бирка (RFID UHF + Temp + Accel)                     │
│  Смарт-ошейник (акустич. датчик + GPS + IMU)               │
│  Сенсор в доильном стакане (проводимость, жир, белок)      │
└─────────────────────────────────────────────────────────────┘
              ↓ LoRa/Wi-Fi/BLE ↓
┌─────────────────────────────────────────────────────────────┐
│      Локальный шлюз (Gateway) на ферме                      │
│  - Raspberry Pi 4 / Industrial Edge Device                  │
│  - LoRa-модем Dragino или SX1308                            │
│  - SD-карта для буфера (кэширование данных)                │
└─────────────────────────────────────────────────────────────┘
              ↓ Internet (MQTT over TLS) ↓
┌─────────────────────────────────────────────────────────────┐
│      Облачная платформа (Yandex Cloud / VK Cloud)          │
├─────────────────────────────────────────────────────────────┤
│  • Backend (FastAPI + Uvicorn)                              │
│  • MQTT Broker (Mosquitto / HiveMQ)                         │
│  • PostgreSQL + TimescaleDB (хранилище)                     │
│  • Redis (кэширование, сессии)                              │
│  • ML Engine (TensorFlow, Scikit-learn)                      │
└─────────────────────────────────────────────────────────────┘
              ↓ REST API ↓
┌─────────────────────────────────────────────────────────────┐
│    Клиентские приложения                                    │
├─────────────────────────────────────────────────────────────┤
│  • Мобильное приложение (React Native)                      │
│  • Веб-интерфейс (Vue.js + TypeScript)                      │
│  • Интеграционные мосты (API для 1С, доильных аппаратов)   │
└─────────────────────────────────────────────────────────────┘
```

#### Ключевые технологии:

| Компонент | Технология | Обоснование |
|-----------|-----------|-------------|
| **Беспроводной протокол (дальние расстояния)** | LoRa (Long Range) | Низкая мощность (10 лет на батарее), дальность до 3 км, работает в сельской местности |
| **Датчик жвачки** | Акустический (микрофон + фильтр) | Более надежен, чем вибрационный; не требует механического контакта |
| **Идентификация животного** | RFID UHF (860–960 МГц) | Пассивное считывание на расстоянии, совместимо с доильными линиями |
| **Измерение качества молока** | Электропроводность (электроды) + IR спектроскопия | Стандартный метод в молочной промышленности, калиброван по ГОСТ 3625-2014 |
| **Обработка сигналов датчиков** | Edge Computing (микроконтроллер STM32) | Фильтрация шума на устройстве, снижение трафика; основная обработка в облаке |
| **Облачная платформа** | Yandex Cloud / VK Cloud Solutions | Расположение на территории РФ (соответствие 152-ФЗ), интеграция с госсервисами |
| **ML-фреймворк** | TensorFlow Lite + Scikit-learn | Компактные модели для edge-устройств, высокая точность для классификации |

#### Интеграция с существующим оборудованием:

- **Доильные установки:** Интеграция через OPC UA (открытый стандарт) с доильными аппаратами Zacharias, Westfalia, Klaus (установка сенсора в стандартный доильный стакан)
- **ERP-системы:** REST API для синхронизации с 1С (модуль «Управление сельскохозяйственным производством»)
- **GPS и системы управления:** Интеграция с системами DairyComp 305, Herd Navigator для обмена данными о животных

#### Патенты и инновационные решения:

1. **Российский патент на метод обнаружения субклинического мастита** (применение комбинации электропроводности молока + поведенческих параметров)
2. **Ноу-хау в области обучения ML-моделей** на локальной популяции коров (адаптация под черно-пеструю и красно-пеструю молочные породы)
3. **Гибридный протокол передачи данных** (LoRa + Wi-Fi + BLE с автоматическим переключением в зависимости от условий покрытия)
4. **Оффлайн-режим работы** с буферизацией данных на локальном хранилище и синхронизацией при восстановлении соединения

7\. Примерный срок окупаемости

Срок окупаемости проекта рассчитывается на основе анализа экономического эффекта от внедрения системы на молочной ферме мощностью 100 голов. Ключевым фактором расчета является текущая рыночная стоимость молока высшего сорта, которая в 2024--2025 годах составляет в среднем 90 рублей за литр (по данным отраслевых отчетов Минсельхоза РФ и порталов Agroinvestor/Milknews).

Базовые показатели фермы: • Поголовье дойного стада: 100 коров. • Средний годовой удой на одну корову: 8 000 литров. • Цена реализации молока (высший сорт): 90 руб./л. • Потенциальная годовая выручка без учета потерь: 100 × 8 000 × 90 = 72 000 000 рублей.

В отсутствие системы непрерывного мониторинга ферма несет существенные финансовые потери, связанные с заболеваниями животных и снижением качества продукции. Согласно статистике, субклинический мастит и другие метаболические нарушения снижают продуктивность на 10--15%, ухудшают сортовые характеристики молока и требуют затрат на лечение.

Структура ежегодных потерь без внедрения системы:

Потери от снижения удоя из-за скрытых заболеваний (субклинический мастит, кетоз). Доля животных, страдающих от подобных состояний в течение года, составляет около 20--30%. Среднее падение надоев оценивается в 10%. Расчетные потери: \~2 500 000 рублей.

Потери от снижения классности молока. Молоко от животных с воспалительными процессами имеет повышенную соматическую клеточность и электропроводность, что приводит к понижению сорта с «Высшего» на «Первый». Разница в закупочной цене составляет порядка 5--7 рублей за литр. Объем молока, теряющего высший сорт, оценивается в 15% от общего объема. Расчетные потери: \~720 000 рублей.

Прямые затраты на ветеринарное лечение и выбраковку. Стоимость лечения одного случая клинического мастита (препараты, работа специалиста) составляет около 15 000 рублей. При частоте заболеваемости 25 случаев в год потери составляют: \~375 000 рублей.

Общие ежегодные потери фермы без системы мониторинга: 2 500 000 + 720 000 + 375 000 = 3 595 000 рублей.

Экономический эффект после внедрения системы: Система позволяет выявлять заболевания на ранней, субклинической стадии, что значительно повышает эффективность лечения и сохраняет продуктивность животных. Ожидаемая эффективность предотвращения тяжелых форм заболеваний и сохранения удоя оценивается в 70--80%.

Восстановление удоя за счет ранней диагностики: 2 500 000 руб. × 70% = 1 750 000 рублей.

Сохранение высшего сорта молока (снижение доли брака по соматике на 80%): 720 000 руб. × 80% = 576 000 рублей.

Экономия на лечении (снижение тяжести заболеваний уменьшает стоимость терапии): 375 000 руб. × 40% = 150 000 рублей.

Итого ежегодная экономия (дополнительный доход): 1 750 000 + 576 000 + 150 000 = 2 476 000 рублей.

Расчет срока окупаемости: Чистый денежный поток в первый год эксплуатации определяется как разница между полученной экономией и годовыми операционными расходами (OPEX). Чистый поток = 2 476 000 руб. (экономия) -- 98 150 руб. (OPEX) = 2 377 850 рублей.

Срок окупаемости капитальных вложений (CAPEX) составляет: 2 815 000 руб. / 2 377 850 руб./год ≈ 1,18 года. В месяцах: 1,18 × 12 ≈ 14,2 месяца.

Таким образом, проект окупается за 14,2 месяца (чуть более одного года). Данный срок является реалистичным для сельскохозяйственных инвестиционных проектов и подтверждает высокую рентабельность технологии даже при консервативных оценках эффективности. Дополнительным преимуществом является социальный эффект: снижение использования антибиотиков, повышение благополучия животных и обеспечение потребителей качественным продуктом.

**12. Безопасность технологии**

Безопасность разработанной системы рассматривается в трех аспектах: физическая безопасность для животных и персонала, информационная безопасность данных и экологическая безопасность материалов.

Физическая безопасность: Все устройства системы (ушная бирка, ошейник) изготавливаются из гипоаллергенных, нетоксичных материалов: медицинского силикона, ABS-пластика и нержавеющей стали марки AISI 304. Конструкция изделий не имеет острых углов и выступающих элементов, что исключает риск травмирования животных или персонала при контакте. Электронные компоненты надежно герметизированы в корпусах со степенью защиты IP67, что предотвращает попадание влаги, пыли и биологических жидкостей внутрь устройства, исключая риск короткого замыкания. Используемые литий-полимерные аккумуляторы оснащены встроенными контроллерами защиты от перегрева, перезаряда и глубокого разряда. Уровень излучения Bluetooth Low Energy (BLE 5.0) находится в пределах безопасных норм и не оказывает негативного влияния на здоровье животных или репродуктивную функцию.

Информационная безопасность: Передача данных от устройств на шлюз и далее в облачное хранилище осуществляется по защищенным каналам связи с использованием протокола шифрования TLS 1.3. Аутентификация пользователей в мобильном приложении и веб-интерфейсе реализована через двухфакторную систему (пароль + одноразовый код), что минимизирует риск несанкционированного доступа. Хранение данных осуществляется на серверах, расположенных на территории РФ, с применением шифрования AES-256. Система соответствует требованиям Федерального закона № 152-ФЗ «О персональных данных», так как обеспечивает конфиденциальность коммерческой информации фермы. Доступ к данным строго разграничен по ролям: фермер видит общую статистику, ветеринар -- медицинские карты животных, администратор -- технические логи системы.

Долгосрочная безопасность: Система не содержит радиоактивных элементов или веществ, способных накапливаться в окружающей среде. Материалы корпусов устойчивы к ультрафиолетовому излучению и не выделяют вредных соединений при длительной эксплуатации. В случае выхода устройства из строя оно не представляет опасности для пищевой цепи, так как не контактирует непосредственно с молоком или мясом (крепление осуществляется на ухе или шее).

**13. Устойчивость реализации проекта и создаваемого продукта**

Устойчивость проекта понимается как совокупность экологической ответственности, экономической жизнеспособности и социальной значимости.

Экологичность материалов и производства: При производстве устройств используются материалы, подлежащие вторичной переработке: термопласты (ABS, PETG) и силикон. Упаковка продукции выполняется из картона, полученного из вторсырья, с отказом от пластиковых вкладышей. Батарейные элементы подлежат обязательной утилизации через партнерские программы с лицензированными компаниями по переработке отходов электроники (например, «Электролом»). Внедрена система take-back: при покупке новых комплектов фермеры могут сдать старые устройства на переработку, получив скидку, что стимулирует замкнутый цикл использования ресурсов.

Минимизация углеродного следа: Локализация производства комплектующих и сборки в России позволяет значительно сократить транспортные расходы и связанные с ними выбросы CO2 по сравнению с импортом готовых решений из Азии или Европы. Логистика внутри страны оптимизирована за счет отгрузки крупными партиями и использования транспорта с высоким коэффициентом загрузки. Энергопотребление самих устройств минимально благодаря использованию энергоэффективных микроконтроллеров и протокола BLE, что позволяет одному комплекту работать от батареи до 2 лет без замены, снижая объем отходов.

Социальная и экономическая устойчивость: Проект способствует повышению продовольственной безопасности страны за счет увеличения эффективности молочного скотоводства. Снижение заболеваемости животных ведет к сокращению применения антибиотиков, что улучшает качество продуктов питания для населения. Для фермеров система обеспечивает стабильный рост доходов и снижение рисков банкротства из-за эпидемий. Создание новых рабочих мест для инженеров, разработчиков и специалистов технической поддержки также вносит вклад в социальную стабильность региона.

**15. Новое программное обеспечение**

Для функционирования системы мониторинга необходимо разработать комплексное программное обеспечение, включающее несколько взаимосвязанных модулей.

Мобильное приложение для фермера (iOS/Android): Предназначено для оперативного контроля состояния стада. Интерфейс включает дашборд с картой фермы, списком животных с цветовой индикацией статуса (норма, внимание, тревога). Основные функции: просмотр индивидуальных показателей коровы (температура, активность, жвачка, удой), получение push-уведомлений о рисках заболеваний (мастит, кетоз, тепловой стресс), доступ к рекомендациям по лечению и кормлению, формирование отчетов по продуктивности стада. Приложение поддерживает экспорт данных в форматы Excel и интеграцию с бухгалтерскими системами (1С).

Серверная часть (Backend): Обеспечивает прием, обработку и хранение больших массивов данных. Использует протоколы MQTT для легковесной передачи телеметрии с устройств и HTTP REST API для взаимодействия с клиентскими приложениями. База данных построена на гибридной архитектуре: PostgreSQL для хранения структурированных данных о животных и пользователях, TimescaleDB для эффективного хранения и анализа временных рядов (показания датчиков). Реализована система триггеров, которые автоматически генерируют события при выходе параметров за установленные пороги.

Модуль машинного обучения (ML Engine): Ядро интеллектуальной системы. Включает модели классификации заболеваний (на базе алгоритмов Random Forest и XGBoost), регрессионные модели для прогнозирования качества молока (LSTM) и алгоритмы кластеризации поведения (K-means) для выявления аномалий. Обучение моделей происходит на исторических данных, собранных в ходе пилотных испытаний. Предусмотрен механизм автоматического переобучения моделей раз в месяц с учетом новых данных для повышения точности прогнозов.

Административная панель: Веб-интерфейс для операторов системы и ветеринарных специалистов. Позволяет управлять учетными записями пользователей, настраивать пороговые значения тревог (например, индивидуальная норма температуры для каждой коровы), просматривать журналы аудита безопасности и управлять параметрами системы.

Интеграционные шлюзы: Программные модули для обмена данными с внешними системами: доильными аппаратами (через протоколы OPC UA или специализированные API производителей), системами управления стадом (DairyComp 305 и аналоги) и облачными платформами (Yandex Cloud, VK Cloud Solutions). Это обеспечивает бесшовную интеграцию системы в существующую IT-инфраструктуру современной фермы.
